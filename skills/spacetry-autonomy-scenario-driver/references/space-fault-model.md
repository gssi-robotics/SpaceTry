- fault subject
    - the environment       (the faults indirectly affect the robot)
        - goals             (e.g., scientific goal object not found, or misplaced)
        - waypoints
        - hazards           (these objects are inserted in the environment according to fault domain, and configured according to fault manifestation)
            - harsh-terrain-tile
            - rock-obstacle
            - hole-tile
            - ...
    - the robot             (the faults directly affect the robot)
        - wheel-1, wheel-2, ...
        - battery
        - sensor
        - engine
        - ...

- fault domain
    - time
        - transient         (one occurrence of the fault over time, for a short duration)
        - permanent         (the fault stays in the simulation for several consecutive time slots)
        - intermittent      (the fault occurs multiple times, occurrences are spread over time)
    - space
        - isolated          (one occurrence of the fault over space, e.g., a single terrain tile)
        - contiguous        (the fault covers multiple contiguous areas, e.g., consecutive tiles of harsh terrain)
        - scattered         (the fault covers multiple scattered areas, e.g., randomly-alternating tiles of harsh and plain terrain)

    - the domain determines
        - space-center-x    (for faults in the environment)
        - space-center-y    (for faults in the environment)
        - space-range       (for faults in the environment)
        - time-range-start  (it can be the entire simulation for environment objects)
        - time-range-end    (it can be the entire simulation for environment objects)

- fault trigger/constraint
    - simulation time       (the injection can only be triggered within a minimum/maximum amount of time)
    - robot attributes      (the injection can only be triggered when robot has a min/max speed, it is within min/max distance from object, etc.)

- fault manifestation
    - stuck                 (the attribute is set to fixed value, which can be zero or "null", e.g., for unavailable sensors)
    - noisy                 (the actual value of the attribute is combined with random errors)
    - degrading             (the actual value of the attribute is combined with error that increases/decreases over space/time)

- attributes
    - every fault subject has one or more attributes targetable by fault injection
    - the attributes can be "read" or "written" by the controller logic
    - examples
        - the robot
            - sensor: reading
            - battery: energy level
            - wheel: rotational speed, wear-out, ...
            - engine: actuator command (inject fuel, ...)
        - the environment
            - harsh-terrain-tile: wheel wear-out rate
            - hole-tile: probability of robot getting stuck
            - rock-obstacle: probability of robot getting stuck
